# AI Styler

Streamlit 기반의 한국 스타일 의류 추천 애플리케이션입니다. 사용자는 보유 중인 옷장을 업로드하고 자동 분류, 보유 의류로 만들 수 있는 코디 추천, 무신사·크림 스타일을 참고한 신규 아이템 추천을 한 곳에서 확인할 수 있습니다.

## 주요 기능
- **나의 옷장**: 이미지 업로드와 메타데이터 입력을 통해 의류를 등록하고 카테고리·스타일 태그를 확인합니다.
- **보유 코디 추천**: 저장된 아이템을 조합하여 색상·스타일·계절 정보를 바탕으로 코디를 제안합니다.
- **추천 아이템(100+)**: 무신사·크림 랭킹/검색 페이지를 실시간 크롤링해 100개 이상 신규 아이템을 제안하고 부족한 카테고리를 자동으로 채워 줍니다.
- **계정 연동(선택)**: 무신사/크림 세션 쿠키를 연결하면 개인화된 추천, 위시리스트 데이터까지 확장할 수 있습니다.

## 폴더 구조
```
AI styler/
├── assets/            # 업로드 이미지 및 샘플 에셋
├── data/              # 인벤토리 DB 및 추천용 카탈로그
├── models/            # 스타일 태깅 및 추천 모델 파일(추후 확장)
├── src/
│   ├── app.py         # Streamlit 진입점
│   └── services/
│       ├── inventory.py
│       ├── recommendations.py
│       ├── marketplace.py
│       └── sources.py  # 무신사·크림 데이터 수집기
└── requirements.txt   # 의존성 목록
```

## 실행 방법
```bash
pip install -r requirements.txt
streamlit run src/app.py
```

Streamlit 실행 후 사이드바에서 **추천 아이템 목표 수 (전체)** 슬라이더로 전체 추천 개수를 조절할 수 있습니다. 기본값은 120개이며 최대 240개까지 노출합니다.

## Musinsa · KREAM 연동

### 계정 연동 (선택)
1. 브라우저에서 무신사 또는 크림에 로그인합니다.
2. 개발자 도구(DevTools) > Application > Cookies에서 `MUSINSA_SESSION`, `k-uid`, `krem_session` 등 세션 쿠키 문자열을 복사합니다.
3. Streamlit 사이드바의 **계정 연동 (선택)** 섹션에 전체 쿠키 문자열을 붙여넣습니다.
4. 저장 즉시 해당 쿠키가 HTTP 세션에 적용되어 개인화된 상품/이미지를 불러옵니다.

쿠키를 입력하지 않아도 공개 랭킹 데이터를 기반으로 100개 이상의 아이템을 추천합니다. 쿠키는 세션 상태에만 저장되며, 파일로 기록되지 않습니다.

### 외부 데이터 수집 정책
- Musinsa: `search.musinsa.com/ranking/best` HTML을 파싱해 이미지, 가격, 상품 링크를 수집합니다.
- KREAM: Next.js 초기 상태(`__NEXT_DATA__`)에서 상품 리스트를 추출합니다.
- 각 요청은 기본적으로 0.3~0.4초 지연을 두어 서비스에 부담을 주지 않도록 합니다.
